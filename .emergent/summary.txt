**Key Technical Concepts**
-   **Role-Based Access Control (RBAC):** Differentiating behavior for 'cashier' vs. other roles, specifically the mandatory shift-opening requirement.
-   **Stateful Resource Locking:** Preventing concurrent use of a shared resource (terminals) by fetching active shifts and disabling occupied options in the UI.
-   **System-Wide UI Blocking:** Using a persistent modal in the root  component, controlled by user role and shift status, to block all app interaction until a condition is met.
-   **Defensive Browser Feature Detection:** Checking for  before attempting to use it to prevent crashes in restricted environments (e.g., Safari private mode).
-   **Hybrid Data Persistence:** Using MongoDB for core application data (bills, tables, terminals) while simultaneously using Supabase for user shifts ( table). This requires careful data synchronization.
