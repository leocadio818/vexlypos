**what currently exists?**
A sophisticated and feature-rich Restaurant POS system has been developed from the ground up. The application consists of a Node.js (Fastify) backend with a Prisma ORM connected to an SQLite database, and a React (Vite) frontend.

**Key implemented features:**
-   **Core Architecture:** Secure user authentication with PINs, a highly granular role-based access control (RBAC) system, and JWT for session management.
-   **Operations:** A dynamic, draggable table map showing real-time table statuses (free, occupied, billed, reserved).
-   **Order Management:** An intuitive order screen with a pop-up numeric keypad, item modifiers, notes, and automatic sending of orders to the kitchen upon leaving the screen or logging out.
-   **Kitchen Display System (KDS):** A dedicated, real-time KDS screen designed for kiosk use, featuring auto-login via a URL parameter and control via a numeric keypad.
-   **Billing & Payments:** A comprehensive billing screen with features for splitting bills and a payment dialog that handles multiple payment methods (including different currencies), calculates change, and processes mixed payments.
-   **Settings & Configuration:** A centralized settings module allows administrators to manage nearly every aspect of the system: users, custom roles, permissions, table layouts, payment methods, sale types (e.g., Dine-in, Take Out), print channels, and station-specific configurations.
-   **Financials:** A dynamic tax configuration module that allows setting up local taxes (like ITBIS and Propina Legal), which are then automatically applied to all bills.
-   **Inventory & Customers:** Modules for managing products (with recipe costs), inventory, suppliers, customers, and a basic loyalty points system.
-   **Reporting & Analytics:** An executive dashboard with KPIs and sales charts.
